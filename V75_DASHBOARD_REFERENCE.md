## CORE PHILOSOPHY

V75 is an algorithmically generated synthetic index. Unlike gold:
- **No macro drivers** — not affected by inflation, yields, DXY, central bank buying
- **No order flow** — no institutional positioning, no COT data
- **No fundamentals** — the price is generated by a cryptographic random number generator

The only meaningful questions:
1. **How is this market behaving statistically?** (regime)
2. **When does it behave best?** (tendency)
3. **What setup has edge right now?** (pattern)
4. **How do YOU perform in these conditions?** (execution)

## PANEL ARCHITECTURE

### Panel A — Market Regime (Phase 1) ✅
**Question it answers**: What state is the market in right now?

**Regime Classifications**:
| Regime | Meaning | Best Strategy Class |
|--------|---------|-------------------|
| Trending | Sustained directional movement | Pullback entries, trend following |
| Ranging | Sideways, compressing ATR | Mean reversion, range fades |
| Expanding | Volatility breakout | Breakout entries, momentum |
| Choppy | No structure, random walk | Sit out or reduce size |

**Indicator Voting System**:
- ADX → Trend strength (>25 = trending, <15 = no trend)
- ATR Ratio → Current ATR / 50-bar ATR (>1.5 = expanding, <0.6 = compressing)
- Hurst Exponent → >0.55 trending, <0.45 mean-reverting, 0.5 = random walk
- Autocorrelation → Return persistence (|>0.3| = trending, |<0.1| = choppy)

**Sub-regimes**:
- trending_up / trending_down / trending_mixed
- compressing / ranging_normal
- breakout_run / volatility_spike

### Panel B — Tendency Engine (Phase 2) ✅
**Question it answers**: When does the market move cleanly?

**Implemented — Statistical Time Profiling Engine**:
- Hour-of-day heatmap: bullish %, volatility, z-score significance per hour (0-23 UTC)
- Day-of-week return distributions with directional bias scoring
- Session-based analysis: Asian, London, New York, Late NY, LDN/NY Overlap
- Monthly seasonal tendency: replaces manual color-coded chart zones with z-score-tested bull/bear scores
- Quarterly cycle analysis: Q1-Q4 profiles + within-quarter phases (early/mid/late) + quarter-boundary shift detection
- Rolling momentum (20/40/60-bar): directional bias, consistency %, strength scoring — replaces manual liquidity projections
- Synthesized summary: current window verdict combining hour + day + session + rolling consensus

### Panel C — Setup Scanner (Phase 2) ✅
**Question it answers**: What setup is active right now?

**Implemented — 5 Setup Detectors with Regime Compatibility Scoring**:
- **Compression Breakout**: BB width percentile tracking → breakout detection via %B + ATR expansion. Stages: compression → breakout.
- **Pullback to Trend**: ADX > 22 + price at 20 EMA zone + wick rejection detection. Stages: approaching → rejection.
- **Mean Reversion**: RSI extreme (< 28 / > 72) + Hurst < 0.5 confirms mean-revert + BB band position.
- **Streak Exhaustion**: Current streak > mean + 1.5σ → momentum fade detection via shrinking returns.
- **Order Block Touch**: Scans for institutional reaction zones (large candle + displacement), detects price return to zone with wick rejection.

**Scoring System**:
- Each setup gets a **confidence score** (0-100) based on pattern quality
- **Regime compatibility** (0-1.0) from a setup↔regime matrix (IDEAL / OKAY / CAUTION)
- **Composite score** = confidence × compatibility — sorted descending, top pick highlighted
- Multi-timeframe scanner: `/api/setups/all` scans all 5 TFs simultaneously

### Panel D — Risk Module (Phase 3) ✅
**Question it answers**: How much should I risk?

**Implemented:**
- ATR-based stop calculator (tight/normal/wide × 1/1.5/2× ATR) with stop-long/stop-short levels
- Position sizing: account balance × risk % ÷ stop distance → lots
- Volatility-adjusted risk scaling: high vol → reduce risk, low vol → slightly increase
- Three-stop comparison strip: see tight/normal/wide side-by-side with lot sizes
- R:R target calculator: 1:1.5, 1:2, 1:3 take-profit levels
- Trade frequency governor: session/daily limits + loss-streak cooldown (green/red light)
- All inputs reactive: balance, risk %, stop type update calculations live

### Panel E — Performance Tracker (Phase 3) ✅
**Question it answers**: Where is YOUR edge?

**Implemented:**
- Trade logging: direction, entry/exit, stop loss, setup type, notes — with regime auto-tagged
- P&L auto-calculated: points, %, R-multiples
- Overall stats: win rate, profit factor, expectancy (R), max drawdown, streaks
- Performance by regime: win rate/PF/expectancy broken down by trending/ranging/etc
- Performance by time: by session (Asian/London/NY), by day of week
- Performance by setup: which detector types are actually working for you
- Equity curve chart: cumulative P&L % with win/loss color-coded points
- Trade log table: full history with delete capability

### Panel F — Alert System (Phase 4) ✅
**Question it answers**: What should you pay attention to now?

**Implemented — 6 Alert Types:**
- Regime shift detected (critical on 1H/4H, warning on lower TFs) with from→to tracking
- Compression forming (BB width percentile ≤15th = breakout conditions building)
- High-performance time window approaching (next hour has statistically significant tendency)
- Overtrading warning (5+ trades in 2 hours = critical discipline guard)
- Streak exhaustion warning (5+ candle same-direction streak)
- Setup confluence (2+ high-confidence setups aligned on same TF)

**Alert Infrastructure:**
- SQLite alert storage with dismiss/history
- Cooldown system prevents spam (per-type, per-TF cooldowns)
- Background scan loop (30s cycle, offset from regime loop)
- Browser notification support (auto-close non-critical after 8s, critical persists)
- Scrollable alert feed with severity color-coding, dismiss per-alert or clear-all
- Alert badge with pulse animation for undismissed count

## TECHNICAL INDICATORS IMPLEMENTED

### ATR (Average True Range)
- Period: 14
- Used for: Volatility measurement, stop sizing, regime classification

### ADX (Average Directional Index)
- Period: 14
- Uses Wilder's smoothing method
- Used for: Trend strength, regime classification

### Hurst Exponent
- Method: Rescaled Range (R/S) analysis
- Max lag: 20
- H > 0.55 = persistent/trending
- H ≈ 0.50 = random walk
- H < 0.45 = anti-persistent/mean-reverting

### Autocorrelation
- Lag: 1 (default)
- Measures return persistence
- Used for: Regime classification, trend validation

### Bollinger Bandwidth
- Period: 20, 2 standard deviations
- Used for: Compression/expansion detection

### Streak Analysis
- Current streak (consecutive same-direction candles)
- Average/max streak length
- Streak standard deviation (for exhaustion detection)

## DERIV API INTEGRATION

### WebSocket Connection
- URL: wss://ws.derivws.com/websockets/v3?app_id={APP_ID}
- Symbol: R_75
- Authentication: API token (read scope)

### Data Requests
- `ticks_history` with `style: "candles"` for historical OHLC
- `ticks` subscription for live tick stream
- Granularity values: 60, 300, 900, 3600, 14400

### API Token Setup
1. Log into Deriv.com
2. Settings → API Token
3. Scopes needed: Read, Trading Information
4. Set as environment variable: DERIV_API_TOKEN

## KEY DESIGN DECISIONS

1. **Voting system for regime**: No single indicator determines regime. Multiple indicators vote, and the highest-scoring regime wins. This prevents false signals from any one measure.

2. **Multi-timeframe regime strip**: Regime on 1m might be "trending" while 4H is "ranging". The strip shows all simultaneously so you can align your trading timeframe with the structural context.

3. **Hurst exponent included**: Most V75 dashboards don't measure this. It's the most direct answer to "is this trending or mean-reverting?" — critical for a synthetic where the answer can change rapidly.

4. **Regime-aware everything**: Every future panel (setups, risk, performance) will be tagged with the regime at that moment. This is the backbone design principle.

5. **Separate server from gold**: Different instrument, different logic, different concerns. No risk of one dashboard affecting the other.
